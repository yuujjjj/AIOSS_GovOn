# PR Review Guide

PR 리뷰는 우선순위가 드러나도록 구조화된 태그를 사용한다.

## 태그 규칙

- `[MUST]`: 병합 전 반드시 수정되어야 하는 문제
- `[SHOULD]`: 권장 수정 사항 또는 품질 개선 의견

## 작성 예시

- `[MUST] 기준 브랜치가 main인데 feature 브랜치 설명이 누락되어 있어 작업 흐름을 검증하기 어렵습니다.`
- `[SHOULD] PR 본문에 테스트 또는 검증 결과를 함께 적어두면 리뷰 속도가 더 빨라집니다.`
- `[SHOULD] 커밋 메시지와 PR 제목을 동일한 Conventional Commits 형식으로 맞추면 이력 추적이 쉬워집니다.`

## 리뷰 원칙

1. 문제를 먼저 적고, 이유를 짧게 설명한다.
2. 수정 방향이 있으면 함께 제안한다.
3. 승인 여부와 별개로 MUST/SHOULD 태그는 일관되게 사용한다.

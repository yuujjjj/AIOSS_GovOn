## 2. 개발 배경

- 공공기관 민원 데이터는 민감한 개인정보를 포함하고 있어 외부 클라우드 전송 시 보안 위반 리스크가 큽니다. 따라서 폐쇄적인 로컬 환경에서의 처리가 필수적입니다.

- 기존의 단순 키워드 분류 방식으로는 복잡한 민원 맥락을 파악하기 어렵습니다.

- 담당 공무원이 수많은 민원 사례를 일일이 확인하는 데 막대한 시간이 소요됩니다. 단순 키워드 매칭이 아닌 의미 기반 검색을 통해 과거 유사 사례를 즉각적으로 참조할 수 있는 시스템이 요구됩니다.

- 대규모 모델 대비 유지비가 저렴하고 특정 도메인에 특화된 sLLM을 활용하여 경제적인 AI 서비스 구현이 가능합니다.

---

## 3. 개발 내용

### 3.1 데이터 확보 및 학습

- 각 지자체별 '전자민원창구' 게시판에서 공개 민원 및 답변 데이터를 수집하는 크롤러를 개발합니다.

- 수집된 데이터를 기반으로 오픈소스 sLLM(Gemma 2, Llama 3 등)에 대한 지도 미세 조정(SFT, Supervised Fine-tuning)을 실시합니다.

### 3.2 표준 답변 생성 엔진

- 민원 답변 표준 서식을 시스템 프롬프트로 설계하여 일관된 톤앤매너를 유지합니다.

- 민원 요지 입력 시, 학습된 데이터를 바탕으로 즉시 고품질의 답변 초안을 생성하는 기능을 구현합니다.

### 3.3 고성능 온프레미스 서빙 최적화

- 폐쇄망 내 설치 및 구동이 가능한 배포 패키징을 진행합니다.

### 3.4 사용자 친화적 UI

- 동서대 디자인학부생과의 융합 캡스톤 협업을 통해 Figma UI/UX 디자인을 진행하고, Figma MCP를 활용하여 React/Next.js 기반의 직관적인 웹 인터페이스를 구축합니다 (민원 입력창, 답변 생성 결과창, 수정 및 복사 기능 등).

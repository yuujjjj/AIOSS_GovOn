## 1. 주요 OSS 라이선스 비교 분석

### 1.1 개요

오픈소스 소프트웨어(OSS) 라이선스는 소프트웨어의 사용, 수정, 배포에 관한 법적 조건을 정의한다. GovOn 프로젝트의 라이선스를 선택하기 위해 대표적인 4개 OSS 라이선스를 비교 분석한다.

### 1.2 각 라이선스의 핵심 특징

#### MIT License

MIT License는 가장 간결하고 허용적인(permissive) 오픈소스 라이선스 중 하나이다. 1988년 매사추세츠 공과대학교(MIT)에서 유래하였으며, 소프트웨어의 자유로운 사용, 복제, 수정, 배포를 허용한다. 유일한 조건은 저작권 고지와 라이선스 문구를 포함하는 것이다. 파생 저작물에 대한 라이선스 제한이 없어 상업적 프로젝트에서도 널리 채택된다.

- **SPDX 식별자**: MIT
- **OSI 승인**: O
- **Copyleft**: 없음
- **대표 프로젝트**: Node.js, React, jQuery, Ruby on Rails

#### Apache License 2.0

Apache Software Foundation이 관리하는 허용적 라이선스이다. MIT License와 유사하게 자유로운 사용을 허용하지만, 명시적 특허 부여 조항(patent grant)을 포함하는 것이 핵심 차이점이다. 기여자가 보유한 특허를 사용자에게 무상으로 실시권을 부여하며, 특허 소송을 제기할 경우 부여된 특허 라이선스가 자동 종료된다(특허 보복 조항). 또한 수정된 파일에 대해 변경 사항을 명시할 것을 요구한다.

- **SPDX 식별자**: Apache-2.0
- **OSI 승인**: O
- **Copyleft**: 없음
- **대표 프로젝트**: Android, TensorFlow, Kubernetes, Apache HTTP Server

#### GNU General Public License 3.0 (GPL 3.0)

자유 소프트웨어 재단(FSF)이 관리하는 강력한 Copyleft 라이선스이다. GPL 3.0으로 배포되는 소프트웨어를 수정하거나 파생 저작물을 만들 경우, 해당 파생 저작물 역시 동일한 GPL 3.0 조건으로 배포해야 한다. 이는 소스 코드의 공개 의무를 수반하며, 독점 소프트웨어와의 결합에 제약을 둔다. 특허 보복 조항과 티보이제이션(Tivoization) 방지 조항도 포함한다.

- **SPDX 식별자**: GPL-3.0-only / GPL-3.0-or-later
- **OSI 승인**: O
- **Copyleft**: 강한 Copyleft (Strong Copyleft)
- **대표 프로젝트**: Linux Kernel(GPL-2.0), GCC, GIMP, WordPress

#### GNU Lesser General Public License 3.0 (LGPL 3.0)

GPL의 변형으로, 라이브러리 사용에 더 유연한 조건을 제공한다. LGPL 라이브러리를 링크하여 사용하는 프로그램은 LGPL 조건을 적용받지 않으나, LGPL 라이브러리 자체를 수정할 경우 해당 수정본은 LGPL로 공개해야 한다. 독점 소프트웨어에서 LGPL 라이브러리를 사용할 수 있다는 점에서 GPL보다 상업적 활용이 용이하다.

- **SPDX 식별자**: LGPL-3.0-only / LGPL-3.0-or-later
- **OSI 승인**: O
- **Copyleft**: 약한 Copyleft (Weak Copyleft)
- **대표 프로젝트**: GNU C Library (glibc), FFmpeg(일부), Qt(이전 버전)

### 1.4 특허 조항 상세 비교

| 항목 | MIT | Apache 2.0 | GPL 3.0 | LGPL 3.0 |
|------|-----|-----------|---------|----------|
| **특허 실시권 명시** | 묵시적 (implicit) | 명시적 부여 (Section 3) | 명시적 부여 (Section 11) | GPL 3.0 준용 |
| **특허 보복 조항** | 없음 | 특허 소송 시 라이선스 종료 (Section 3) | 특허 소송 시 라이선스 종료 | GPL 3.0 준용 |
| **기여자 특허 범위** | 규정 없음 | 기여 내용에 한정 | 전체 저작물 범위 | GPL 3.0 준용 |

MIT License는 특허에 대한 명시적 규정이 없으므로, 특허가 중요한 프로젝트에서는 Apache 2.0이 더 안전한 선택이 될 수 있다. 다만 GovOn과 같은 학술/교육 목적 프로젝트에서는 MIT의 단순성이 더 큰 장점으로 작용한다.

### 1.5 라이선스 호환성 매트릭스

하나의 프로젝트에서 서로 다른 라이선스의 구성 요소를 결합할 때, 라이선스 간 호환성이 중요하다.

| 결합 대상 | MIT | Apache 2.0 | GPL 3.0 | LGPL 3.0 |
|-----------|:---:|:----------:|:-------:|:--------:|
| **MIT** | O | O | O (GPL 적용) | O |
| **Apache 2.0** | O | O | O (GPL 적용) | O |
| **GPL 3.0** | O (GPL 적용) | O (GPL 적용) | O | O (GPL 적용) |
| **LGPL 3.0** | O | O | O (GPL 적용) | O |

- MIT 코드는 GPL 프로젝트에 포함 가능하지만, 결합물 전체에 GPL이 적용된다.
- Apache 2.0은 GPL 2.0과 호환되지 않으나, GPL 3.0과는 호환된다.
- LGPL 라이브러리는 링크 방식으로 MIT/Apache 프로젝트에서 사용 가능하다.

## 2. EXAONE 라이선스 호환성 분석

### 2.1 LGAI EXAONE License 개요

EXAONE-Deep-7.8B 모델은 LG AI Research가 개발하였으며, **LGAI EXAONE AI Model License Agreement**에 따라 배포된다. 이 라이선스는 OSI 승인을 받은 표준 오픈소스 라이선스가 아닌, LG AI Research가 자체적으로 정의한 사용자 정의 라이선스(custom license)이다.

#### 허용 사항

| 구분 | 내용 |
|------|------|

|------|------|

#### 상업적 사용 기준

EXAONE 라이선스는 상업적 사용을 완전히 금지하지 않으나, 일정 규모 이상의 상업적 사용에는 별도 계약을 요구한다. 구체적인 기준은 다음과 같다.

- **소규모 상업적 사용**: 연간 매출 기준(라이선스 버전별 상이)에 따라 허용될 수 있음
- **대규모 상업적 사용**: LG AI Research와의 별도 라이선스 계약 필요
- **학술/연구 사용**: 제한 없이 허용

> **주의**: EXAONE 라이선스 조건은 버전에 따라 변경될 수 있으므로, 반드시 해당 모델 배포 페이지의 최신 라이선스 전문을 확인해야 한다.

### 2.3 MIT License와의 호환성 검토

GovOn 프로젝트 소스 코드(MIT)와 EXAONE 모델 가중치(EXAONE License)는 **별도의 저작물(separate works)** 로서 공존한다. 호환성 분석은 다음과 같다.

#### 구성 요소별 라이선스 적용 범위

```
GovOn 프로젝트 구조와 라이선스 적용
===========================================

[MIT License 적용 영역]
  src/inference/api_server.py        -- 소스 코드
  src/inference/retriever.py         -- 소스 코드
  src/inference/schemas.py           -- 소스 코드
  src/training/                      -- 학습 스크립트
  src/data_collection_preprocessing/ -- 데이터 파이프라인
  ...

[EXAONE License 적용 영역]
  models/exaone-deep-7.8b/           -- 모델 가중치
  models/exaone-deep-7.8b-awq/       -- 양자화 가중치
  models/finetuned/                   -- 파인튜닝 가중치 (파생 모델)

[공공데이터 이용 정책 적용 영역]
  data/raw/                           -- 원본 공공데이터
  data/processed/                     -- 전처리된 학습 데이터
```

#### 호환성 판단

| 검토 항목 | 결과 | 근거 |
|-----------|------|------|
| 소스 코드와 모델 가중치 분리 가능 여부 | O | 소스 코드는 모델과 독립적으로 동작 가능한 구조 |
| MIT 코드가 EXAONE 조건에 영향 받는지 | X | 별도 저작물로 라이선스가 독립적으로 적용 |
| EXAONE 모델이 MIT 조건에 영향 받는지 | X | 모델 가중치는 MIT 라이선스 범위 밖 |
| 통합 배포 시 라이선스 충돌 여부 | 조건부 | 각 구성 요소의 라이선스를 명확히 구분하여 고지 필요 |

**결론**: MIT License와 EXAONE License는 적용 대상이 다르므로(소스 코드 vs. 모델 가중치) 직접적인 호환성 충돌은 발생하지 않는다. 다만 프로젝트를 통합 배포할 때 각 구성 요소에 적용되는 라이선스를 명확히 구분하여 고지해야 한다.

### 2.4 모델 가중치 배포 시 주의사항

GovOn은 EXAONE-Deep-7.8B를 QLoRA 파인튜닝 후 AWQ 양자화하여 사용한다. 이 과정에서 생성되는 파생 모델에 대한 라이선스 의무는 다음과 같다.

| 파생물 유형 | 라이선스 적용 | 배포 시 의무 |
|-------------|--------------|-------------|
| QLoRA 어댑터 (LoRA 가중치만) | EXAONE License 적용 가능성 높음 | 원본 모델 라이선스 조건 명시 필요 |
| LoRA 병합 가중치 | EXAONE License 적용 | 원본 라이선스 전문 포함, 상업적 사용 제한 고지 |
| AWQ 양자화 가중치 | EXAONE License 적용 | 원본 라이선스 전문 포함, 양자화에 의한 변경 사항 명시 |
| FAISS 인덱스 파일 | 학습 데이터 라이선스 적용 | 공공데이터 이용 조건 준수 |

**실무 권고사항**:

1. **모델 가중치는 소스 코드 저장소(GitHub)에 포함하지 않는다.** Git LFS나 별도 모델 저장소(Hugging Face Hub 등)를 활용한다.
2. **README 또는 MODEL_CARD에 모델 라이선스를 별도로 명시한다.** "이 프로젝트의 소스 코드는 MIT License이나, 모델 가중치는 EXAONE License를 따릅니다"와 같은 문구를 포함한다.
3. **파인튜닝 가중치 배포 시**, 원본 EXAONE 라이선스 전문을 함께 제공하고, 파생 모델임을 명시한다.
4. **상업적 배포 검토 시**, LG AI Research에 별도 라이선스 계약 필요 여부를 사전에 확인한다.

## 3. 공공데이터 라이선스 검토

### 3.1 한국 공공데이터포털 이용 정책

#### 법적 근거

한국의 공공데이터 이용은 **공공데이터의 제공 및 이용 활성화에 관한 법률**(공공데이터법, 2013년 제정)에 근거한다. 이 법률은 공공데이터의 민간 활용을 촉진하기 위한 법적 틀을 제공한다.

#### 공공데이터 이용 원칙

| 원칙 | 내용 |
|------|------|
| 무상 제공 원칙 | 공공데이터는 원칙적으로 무상 제공 (공공데이터법 제31조) |
| 자유로운 이용 | 영리/비영리 목적 모두 이용 가능 |
| 제한적 예외 | 개인정보, 국가안보, 제3자 권리 침해 데이터는 제외 |
| 2차 가공 허용 | 분석, 융합, 활용 서비스 개발에 사용 가능 |

#### 공공데이터포털(data.go.kr) 이용 조건

공공데이터포털에서 제공하는 데이터는 **공공누리(Korea Open Government License)** 유형에 따라 이용 조건이 달라진다.

| 공공누리 유형 | 조건 | 상업적 이용 | 2차 가공 |
|---------------|------|:-----------:|:--------:|
| **제1유형** (출처표시) | 출처 표시 | O | O |
| **제2유형** (출처표시 + 상업적 이용 금지) | 출처 표시, 비상업적 사용만 | X | O |
| **제3유형** (출처표시 + 변경 금지) | 출처 표시, 원형 유지 | O | X |
| **제4유형** (출처표시 + 상업적 이용 금지 + 변경 금지) | 출처 표시, 비상업적, 원형 유지 | X | X |

GovOn 프로젝트에서 활용하는 민원 데이터는 대부분 **제1유형(출처표시)** 에 해당하며, 출처를 명시하는 조건으로 자유롭게 활용할 수 있다.

### 3.2 AI Hub 데이터 이용 조건

#### AI Hub 개요

AI Hub(aihub.or.kr)는 한국지능정보사회진흥원(NIA)이 운영하는 AI 학습용 데이터 플랫폼이다. GovOn은 민원 분석을 위한 학습 데이터를 AI Hub에서 수집한다.

#### AI Hub 데이터 이용 규정

| 항목 | 내용 |
|------|------|
| 이용 자격 | 회원가입 후 데이터 이용 신청 (승인 필요) |
| 이용 목적 | AI 연구/개발 목적 (학술, 산업 포함) |
| 상업적 이용 | 데이터셋별 상이 (개별 이용약관 확인 필요) |
| 재배포 금지 | 원본 데이터의 직접 재배포 금지 |
| 2차 가공물 | 가공된 결과물은 이용 가능 (개별 조건 확인) |
| 출처 표시 | 의무 (AI Hub 및 원 데이터 제공 기관 명시) |
| 이용 기간 | 신청 시 명시한 기간 (연장 가능) |

#### AI Hub 데이터 사용 시 준수사항

1. **개인정보 보호**: 데이터에 포함된 개인정보는 비식별화(PII 마스킹) 처리 후 사용해야 한다.
2. **목적 외 사용 금지**: 신청 시 명시한 목적 외에 데이터를 사용할 수 없다.
3. **출처 표시**: 학술 논문, 보고서, 서비스에 "AI Hub" 및 원 데이터 제공 기관을 명시해야 한다.
4. **데이터 반환/삭제**: 이용 기간 종료 시 보유 데이터를 삭제하거나 반환해야 한다.

### 3.3 학습 데이터 재배포 가능 여부

GovOn의 데이터 파이프라인에서 처리되는 데이터의 재배포 가능 여부를 정리하면 다음과 같다.

| 데이터 유형 | 원본 재배포 | 가공물 배포 | 조건 |
|-------------|:-----------:|:-----------:|------|
| 공공데이터포털 데이터 (제1유형) | O | O | 출처 표시 |
| AI Hub 학습 데이터 | X | 조건부 | 개별 이용약관 확인, 목적 외 사용 금지 |
| PII 마스킹 처리된 데이터 | X (원본 기준 적용) | 조건부 | 원본 데이터 라이선스 조건 준수 |
| FAISS 인덱스 (임베딩) | 해당 없음 | 조건부 | 원본 데이터 라이선스 및 임베딩 모델 라이선스 확인 |
| 파인튜닝용 변환 데이터 | X | 조건부 | 원본 데이터 이용 조건 준수 |

**실무 권고사항**:

1. **원본 학습 데이터는 GitHub 저장소에 포함하지 않는다.** 데이터 수집/전처리 스크립트만 공개한다.
2. **데이터 출처를 README 또는 DATA_SOURCES.md에 명시한다.** 공공데이터포털 및 AI Hub 출처 URL을 기재한다.
3. **개인정보가 포함될 수 있는 민원 데이터는 반드시 PII 마스킹을 적용한 후 처리한다.** GovOn은 `src/data_collection_preprocessing/` 모듈에서 이를 수행한다.
4. **데이터 이용 기간 관리를 위한 기록을 유지한다.** 어떤 데이터를 언제 다운로드하여 어떤 목적으로 사용했는지 추적 가능해야 한다.

### 4.2 EXAONE 모델 라이선스와의 공존 방안

GovOn은 MIT 소스 코드와 EXAONE 모델 가중치가 공존하는 **이중 라이선스 구조(dual licensing structure)** 를 채택한다.

#### 공존 구조 설계

```
프로젝트 라이선스 구조
===========================================

  [전체 프로젝트]
    |
    +-- [소스 코드] -----> MIT License
    |     LICENSE 파일에 명시
    |
    +-- [모델 가중치] ---> EXAONE License
    |     models/LICENSE 또는 MODEL_CARD.md에 명시
    |
    +-- [학습 데이터] ---> 공공데이터 이용 정책
          DATA_SOURCES.md에 출처 및 이용 조건 명시
```

#### 라이선스 고지 방안

1. **루트 LICENSE 파일**: MIT License 전문 (소스 코드에 적용)
2. **README.md 라이선스 섹션**: 각 구성 요소별 라이선스를 구분하여 명시
3. **모델 관련 고지**: MODEL_CARD.md 또는 models/LICENSE에 EXAONE License 전문 포함
4. **데이터 출처 고지**: DATA_SOURCES.md에 공공데이터 출처 및 이용 조건 명시
5. **NOTICE 파일**: 서드파티 라이선스 고지 통합 (사용하는 OSS 라이브러리 목록)

#### README.md 라이선스 섹션 예시

```markdown
## License

- **소스 코드**: [MIT License](LICENSE)
- **EXAONE 모델 가중치**: [EXAONE AI Model License](https://huggingface.co/LGAI-EXAONE/EXAONE-Deep-7.8B)
  - 모델 가중치는 본 저장소에 포함되지 않으며, 별도로 다운로드해야 합니다.
  - 상업적 사용 시 LG AI Research의 라이선스 조건을 확인하세요.
- **학습 데이터**: 공공데이터포털 및 AI Hub 이용 정책에 따릅니다.
```

### 4.3 향후 라이선스 변경 시 고려사항

프로젝트 성장에 따라 라이선스 변경이 필요할 수 있는 시나리오와 고려사항을 정리한다.

#### 라이선스 변경 가능 시나리오

| 시나리오 | 현재 (MIT) | 변경 후보 | 고려사항 |
|----------|-----------|-----------|----------|
| 특허 보호 필요 시 | 묵시적 특허만 | Apache 2.0 | 기존 MIT 코드와 호환, 특허 보복 조항 추가 |
| 상업적 파생물 제한 필요 시 | 제한 없음 | AGPL 3.0 | 기존 기여자 동의 필요, SaaS 소스 공개 의무 발생 |
| 기업 채택 촉진 시 | 유지 | MIT 유지 | 현재 라이선스가 가장 적합 |
| EXAONE 라이선스 변경 시 | 독립적 | 상황에 따라 | 모델 라이선스 변경 시 프로젝트 구조 재검토 필요 |

#### 라이선스 변경 시 필수 절차

1. **기존 기여자 동의**: 모든 기여자로부터 라이선스 변경에 대한 동의를 받아야 한다. CLA(Contributor License Agreement) 미체결 시 동의 확보가 어려울 수 있다.
2. **호환성 검증**: 변경 대상 라이선스가 현재 사용 중인 서드파티 라이선스와 호환되는지 검증한다.
3. **고지 업데이트**: LICENSE 파일, README, NOTICE, 각 소스 파일의 라이선스 헤더를 일괄 업데이트한다.
4. **법적 검토**: 라이선스 변경의 법적 유효성을 전문가에게 검토받는 것을 권장한다.

#### 권고: CLA 도입 검토

향후 외부 기여자가 늘어날 경우, CLA(Contributor License Agreement)를 도입하면 라이선스 변경 유연성을 확보할 수 있다. CLA는 기여자가 자신의 기여에 대해 프로젝트 관리자에게 충분한 권한을 부여하는 계약이다.

# ADR-001: EXAONE-Deep-7.8B 모델 선정

## Status

Accepted

## Context

GovOn은 지방자치단체 민원 분석 및 답변 초안 생성을 위한 온프레미스 AI 시스템이다. 기반 LLM을 선정할 때 다음 제약 조건을 충족해야 한다.

1. **한국어 민원 도메인 성능**: 민원 분류, 관련 법령 인용, 답변 초안 생성 등 한국어 공공행정 텍스트를 정확히 처리해야 한다.
2. **폐쇄망 배포**: 공공기관 보안 정책상 외부 API 호출이 불가하므로, 모델 가중치를 내부 서버에 배포할 수 있어야 한다.
3. **GPU VRAM 제한**: 단일 GPU(16~24GB VRAM) 환경에서 양자화를 통해 서빙 가능한 파라미터 규모여야 한다.
4. **라이선스**: 상업적 사용 및 기관 내 배포가 가능한 오픈소스 라이선스가 필요하다.
5. **파인튜닝 용이성**: QLoRA 등 효율적 파인튜닝 기법이 적용 가능해야 하며, Hugging Face 생태계와 호환되어야 한다.

## 검토 후보

| 후보 | 파라미터 | 한국어 성능 | 라이선스 | 양자화 호환 | 비고 |
|------|----------|------------|----------|------------|------|
| **EXAONE-Deep-7.8B** | 7.8B | 우수 (한국어 사전학습 데이터 대규모 포함, 추론 체인 지원) | Apache 2.0 | AWQ, GPTQ, BnB 모두 지원 | LG AI Research 개발, `<thought>` 태그 기반 추론 체인 |
| Qwen2.5-7B | 7B | 양호 (다국어 모델, 한국어는 부차적) | Apache 2.0 | AWQ, GPTQ 지원 | 중국어 중심 다국어 모델 |
| Llama-3-8B-Korean | 8B | 보통 (영어 기반 모델에 한국어 추가 학습) | Llama 3 Community License | AWQ, GPTQ 지원 | Meta 기반 커뮤니티 한국어 적응 |

### 상세 비교

**EXAONE-Deep-7.8B**
- LG AI Research가 한국어를 포함한 대규모 코퍼스로 사전학습한 모델이다.
- `<thought>` 태그를 활용한 추론 체인(chain-of-thought) 기능이 내장되어, 민원 분류 근거를 단계적으로 생성할 수 있다.
- `trust_remote_code=True`로 커스텀 모델 코드를 로드하며, transformers 버전에 따라 런타임 패치가 필요할 수 있다.
- Apache 2.0 라이선스로 공공기관 배포에 법적 제약이 없다.

**Qwen2.5-7B**
- 다국어 성능이 우수하나, 한국어 공공행정 도메인에서는 EXAONE 대비 미세 조정 후에도 법령 인용 정확도가 낮았다.
- 한국어 토크나이저 효율이 EXAONE보다 떨어져 동일 문맥 길이 대비 처리 가능한 한국어 텍스트 양이 적다.

**Llama-3-8B-Korean**
- 영어 기반 모델에 한국어 코퍼스를 추가 학습한 구조로, 한국어 생성 품질이 네이티브 한국어 모델 대비 부자연스러운 경우가 있었다.
- Llama 3 Community License는 Apache 2.0보다 사용 조건이 제한적이며, 월간 활성 사용자 7억 명 이상 시 별도 협의가 필요하다(공공기관에는 해당하지 않으나 라이선스 검토 부담 존재).

## Decision

**EXAONE-Deep-7.8B**를 기반 모델로 선정한다.

핵심 근거는 다음과 같다.

1. **한국어 특화 아키텍처**: 한국어 사전학습 데이터 비중이 높아 민원 도메인 파인튜닝 시 수렴 속도와 최종 품질이 우수하다.
2. **추론 체인 내장**: `<thought>` 태그 기반 단계적 추론이 민원 분류 근거 제시에 적합하다. 공무원이 AI 답변의 논리 과정을 확인할 수 있어 신뢰성을 높인다.
3. **폐쇄망 배포 무제약**: Apache 2.0 라이선스로 기관 내 자유로운 배포와 수정이 가능하다.
4. **양자화 파이프라인 검증 완료**: QLoRA(NF4) 파인튜닝 후 AWQ W4A16g128 양자화까지 전체 파이프라인이 검증되었다.
5. **적정 파라미터 규모**: 7.8B 파라미터는 AWQ 양자화 시 약 4~5GB VRAM으로 서빙 가능하여 단일 GPU 제약을 충족한다.

## Consequences

### 긍정적 영향

- 한국어 민원 텍스트에 대한 높은 기본 이해도로, 파인튜닝 데이터량을 줄이면서도 양질의 결과를 얻을 수 있다.
- `<thought>` 태그 기반 추론 과정이 민원 답변의 설명 가능성(explainability)을 확보한다.
- Apache 2.0 라이선스로 공공기관 조달 및 감사 과정에서의 법적 리스크가 없다.
- Hugging Face Hub에 공식 모델이 등록되어 있어 모델 관리가 용이하다.

### 부정적 영향

- EXAONE 모델의 커스텀 코드(`trust_remote_code=True`)가 transformers 버전 업데이트 시 호환성 문제를 일으킬 수 있다. 실제로 `get_input_embeddings`, `get_output_embeddings` 등의 몽키패치가 필요한 상황이 발생했다(참고: `src/training/train_qlora.py`, `src/inference/vllm_stabilizer.py`).
- EXAONE 모델 생태계가 Llama/Qwen 대비 작아, 커뮤니티 지원(튜토리얼, 이슈 해결 사례 등)이 제한적이다.
- 향후 모델 업데이트 주기가 LG AI Research 일정에 의존한다.

### 향후 고려사항

- transformers 메이저 버전 업데이트 시 `vllm_stabilizer.py`의 런타임 패치 호환성을 점검해야 한다.
- EXAONE 후속 모델(파라미터 규모 확대 등) 출시 시, 양자화 파이프라인 재검증과 함께 모델 교체를 검토한다.
- 다국어 민원 지원이 요구될 경우, 다국어 모델로의 전환 또는 병행 운영을 재평가한다.

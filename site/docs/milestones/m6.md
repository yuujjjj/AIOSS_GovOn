# M6: CI/CD 고도화

M6의 목표는 자동화 개수를 늘리는 것이 아니라, 실제 MVP surface에 맞는 품질 게이트를 세우는 것이다.

## 현재 반영된 방향

- `PR Gate` 단일 게이트 중심 구조
- shell-first runtime contract를 Playwright로 검증
- docs strict build를 PR 단계로 이동
- Docker publish와 demo deploy 분리
- auto deploy 대신 manual Cloud Run verification 사용

## 남은 작업

- 저장소 branch protection에 required checks 연결
- `govon` 설치 경험이 완성되면 packaging acceptance 추가
- approval-gated shell UI까지 포함하는 상위 E2E 확장

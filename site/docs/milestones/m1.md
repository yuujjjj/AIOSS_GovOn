# M1: 기획 및 설계

M1은 GovOn의 제품 방향을 "웹 서비스"가 아니라 "shell-first local runtime"으로 고정하는 단계였다.

## 산출물

- CLI + daemon + approval loop 구조 정의
- session persistence를 SQLite 기반으로 정리
- R1 범위와 post-R1 범위 분리
- 아키텍처 ADR 작성

## 현재 의미

이 마일스톤에서 정한 기준 덕분에, 이후 CI/CD도 web surface가 아니라 shell runtime contract를 중심으로 재정렬할 수 있다.
